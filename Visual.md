# Visual.md — UX/UI 다이어그램

사용자가 이 웹사이트를 어떻게 쓰게 될지, 화면 흐름과 각 화면의 구성을 ASCII 아트로 시각화한 문서입니다.

---

## 1. 전체 화면 흐름도 (User Flow)

```
 ┌──────────────────────────────────┐
 │  웹사이트 접속 (localhost:3000)  │
 └─────────────┬────────────────────┘
               │
               ▼
       [로그인 되어 있나?]
         │           │
       아니오         예
         │           │
         ▼           ▼
 ┌───────────────┐  ┌───────────────────────┐
 │  /login 화면  │  │ /docs 문서 목록 화면  │
 │  (이름 입력)  │  └───────────┬───────────┘
 └───────┬───────┘              │
         │                      │
         │ 로그인 성공           │
         └──────────────────────┤
                                │
                                ▼
                   [무엇을 할까?]
                     │        │
                     │        │
              새 문서│         │기존 문서
                     │        │
                     ▼        ▼
            ┌─────────────────────────┐
            │ /docs/[id] 편집 화면    │
            │ (실시간 협업 에디터)    │
            └─────────────────────────┘
                     │
                     │ 뒤로
                     ▼
                  /docs
```

### 2.3. 문서 편집 화면 (`/docs/[id]`) — 핵심 화면

```
 ┌─────────────────────────────────────────────────────────┐
 │  [← 문서 목록]                                          │
 │                                                         │
 │  제목: ┌─────────────────────────────────────────┐     │
 │        │ 4월 정기 회의                            │     │
 │        └─────────────────────────────────────────┘     │
 │                                                         │
 │  접속 중:  🟢 Alice   🟡 Bob   🔵 Charlie              │
 │                                                         │
 ├─────────────────────────────────────────────────────────┤
 │  [B] [I] [H1] [H2] [•] [1.] [↶] [↷]   ← 툴바           │
 ├─────────────────────────────────────────────────────────┤
 │                                                         │
 │   # 오늘 회의 안건                                      │
 │   ──────────────                                       │
 │                                                         │
 │   1. 프로젝트 진행 상황 공유                            │
 │      - 디자인 80% 완료│(←Alice 커서)                   │
 │      - 개발 40% 진행 중                                │
 │                                                         │
 │   2. 다음 스프린트 계획                                 │
 │      - 4월 15일 출시 목표│(←Bob 커서)                  │
 │                                                         │
 │   3. 블로커 논의                                        │
 │                   │(←Charlie 커서)                      │
 │                                                         │
 │                                                         │
 │                                                         │
 │                                                         │
 └─────────────────────────────────────────────────────────┘
   💾 자동 저장됨
```

> **포인트**: 각 사용자의 커서는 상단 presence 배지와 **같은 색**으로 표시됩니다. 커서 위에 이름 라벨도 뜹니다.

## 3. 실시간 협업 동작 방식 (Data Flow)

### 3.1. 두 명이 동시 편집하는 순간

```
   [Alice 브라우저]              [Bob 브라우저]
   ┌──────────────┐              ┌──────────────┐
   │ 안녕하세 │   │              │ 안녕하세 │   │
   │ 요!     ↑    │              │ 요!     ↑    │
   │      Alice   │              │      Alice   │
   │              │              │      │       │
   │              │              │      Bob     │
   └──────┬───────┘              └──────┬───────┘
          │                             │
          │  (타이핑)                    │  (타이핑)
          ▼                             ▼
      ┌────────────────────────────────────┐
      │   WebSocket 서버 (port 1234)       │
      │   = 실시간 중계소                    │
      │   - 변경사항을 모두에게 전달         │
      │   - 2초마다 DB에 저장               │
      └────────────┬───────────────────────┘
                   │
                   ▼
            ┌─────────────┐
            │  SQLite DB  │
            │  (dev.db)   │
            └─────────────┘
```

### 3.2. 타이핑 한 글자가 전달되는 과정

```
   Alice가 "가" 입력
           │
           ▼
   ┌───────────────────┐
   │ 브라우저의 Y.Doc   │ ← CRDT 변경 생성
   └────────┬──────────┘
            │
            │ (WebSocket으로 전송)
            ▼
   ┌───────────────────┐
   │  중계 서버        │
   │  (y-websocket)    │
   └────────┬──────────┘
            │
            ├──────────────────────┐
            │                      │
            ▼                      ▼
   ┌───────────────┐     ┌──────────────────┐
   │ Bob 브라우저   │     │ Charlie 브라우저 │
   │ "가" 즉시 표시  │     │ "가" 즉시 표시   │
   └───────────────┘     └──────────────────┘

            │
            │ (2초 후 debounce)
            ▼
   ┌───────────────────┐
   │   SQLite에 저장   │
   │  ydocState: bytes │
   └───────────────────┘
```

```
       └──────┬───────┘  └──────┬───────┘  │
              │                 │          │
              └────────┬────────┘          │
                       │                   │
                       ▼                   │
               ┌───────────────┐           │
               │  SQLite DB    │           │
               │  dev.db       │           │
               │               │           │
               │  ┌─────────┐  │           │

               │  │Document │  │           │

               └───────────────┘           │
                                           │
                                           ▼
                                   (모두 로컬 PC 내에서 동작)
```

## 5. 상태별 UI 변화 (State Variations)

### 5.1. 접속자가 1명일 때

```
 접속 중:  🟢 Alice
```

### 5.2. 접속자가 여러 명일 때

```
 접속 중:  🟢 Alice   🟡 Bob   🔵 Charlie   🟣 Dave
```

### 5.3. WebSocket 연결이 끊겼을 때

```
 ⚠️ 오프라인 — 재연결 시도 중...
 (편집은 계속 가능, 재연결 시 자동 동기화)
```

### 5.4. 저장 상태 표시

```
   정상 저장:    💾 자동 저장됨
   저장 중:      ⏳ 저장 중...
   오류:         ⚠️ 저장 실패 (재시도 중)
```

## 6. 색상 규칙 (Color System)

각 사용자는 이름 해시로 **프리셋 색상 10개 중 하나**를 자동 할당받습니다. 이 색상은 3군데에서 동일하게 사용됩니다:

```
  ┌─────────────────────────────────────────┐
  │  1. 상단 presence 배지의 동그라미        │
  │     🟢 Alice                            │
  │                                         │
  │  2. 에디터 본문 속 Alice의 커서          │
  │     ...내용 여기│ (초록 커서)            │
  │                                         │
  │  3. 커서 위 이름 라벨                    │
  │     ┌──────┐                            │
  │     │Alice │  ← 초록 배경                │
  │     └───┬──┘                            │
  │         │                               │
  │     ...여기│                            │
  └─────────────────────────────────────────┘
```

프리셋 팔레트 예시:
```
  🔴 #EF4444   🟠 #F97316   🟡 #EAB308
  🟢 #22C55E   🟢 #10B981   🔵 #3B82F6
  🔵 #06B6D4   🟣 #8B5CF6   🟣 #A855F7
  🟤 #F43F5E
```

## 7. 반응형 레이아웃 (데스크톱 우선)

MVP는 데스크톱 기준으로 설계합니다. 모바일은 기본 동작만 보장하고 세부 최적화는 post-MVP.

```
   [데스크톱 1280px+]
   ┌────────────────────────────────────────┐
   │  상단바 (풀 너비)                       │
   ├────────────────────────────────────────┤
   │                                        │
   │      에디터 (최대 너비 800px 중앙정렬)  │
   │      ┌──────────────────────┐         │
   │      │                      │         │
   │      │                      │         │
   │      └──────────────────────┘         │
   │                                        │
   └────────────────────────────────────────┘

   [모바일]
   ┌──────────┐
   │ 상단바    │
   ├──────────┤
   │          │
   │ 에디터    │
   │ (풀 너비) │
   │          │
   └──────────┘
```
