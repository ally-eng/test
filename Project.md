# Project.md — 실시간 문서 협업 시스템

## 1. 프로젝트 개요

Google Docs처럼 **여러 사람이 하나의 문서를 동시에 편집할 수 있는 웹 기반 협업 도구**를 만듭니다. 첫 버전은 "동작하는 최소 기능(MVP)"에 집중하여, 실시간 동시 편집이 실제로 된다는 것을 빠르게 검증하는 것이 목표입니다.

- **프로젝트 이름**: 실시간 문서 협업 시스템 (MVP)
- **형태**: 웹 애플리케이션 (브라우저에서 접속)
- **대상 사용자**: 팀 단위로 문서를 함께 작성해야 하는 소규모 그룹
- **현재 단계**: MVP (Minimum Viable Product)

---

## 2. 해결하고자 하는 문제

| 기존 불편함 | 이 프로젝트가 주는 해결책 |
|---|---|
| 워드 파일을 메일/메신저로 주고받으며 버전이 꼬임 | 한 문서를 여러 명이 **동시에** 편집 |
| 누가 어디를 편집 중인지 알 수 없음 | **실시간 커서 + 이름 라벨** 표시 |
| 누가 지금 문서를 보고 있는지 모름 | 상단에 **접속 중인 사용자 목록** 표시 |
| 편집 충돌로 내용 유실 | Yjs CRDT가 **충돌을 자동 해결**, 유실 없음 |

---

## 3. 목표 (Goals)

### 1차 목표 (MVP에서 반드시 달성)
1. 여러 사용자가 같은 문서를 동시에 편집할 수 있다
2. 타이핑하면 1초 이내에 다른 사용자 화면에 반영된다
3. 동시 편집 시 글자 손실이나 중복이 발생하지 않는다
4. 다른 사용자의 커서 위치와 이름이 색깔로 표시된다
5. 문서를 닫고 다시 열어도 내용이 그대로 복원된다

### 2차 목표 (MVP 이후)
- 문서별 권한 관리
- 댓글, 버전 히스토리
- 다양한 로그인 방법 (구글, 카카오 등)
- 실제 서비스 수준의 배포

---

## 4. 기능 정의

### ✅ MVP에 포함되는 기능

| # | 기능 | 설명 |
|---|---|---|
| F1 | 간편 로그인 | 이름만 입력하면 로그인되는 방식 (비밀번호 X) |
| F2 | 문서 목록 보기 | 전체 문서 리스트와 마지막 수정 시간 표시 |
| F3 | 새 문서 생성 | 버튼 하나로 빈 문서 생성 |
| F4 | 문서 편집 | 제목 수정 + 본문 리치 텍스트 편집 (굵게/기울임/제목/목록) |
| F5 | 실시간 동시 편집 | 여러 사람이 동시에 타이핑, CRDT로 충돌 자동 해결 |
| F6 | 실시간 커서 표시 | 다른 사용자의 커서 위치를 고유 색상과 이름 라벨로 표시 |
| F7 | 접속 인원 표시 | 현재 문서를 열고 있는 사용자 목록을 상단바에 표시 |
| F8 | 문서 자동 저장 | 편집 내용이 2초마다 자동 저장, 새로고침해도 유지 |
| F9 | URL로 문서 공유 | 문서 URL을 복사해서 다른 사람에게 전달 가능 |
| F10 | 로그아웃 | 세션 쿠키 삭제 |

### ❌ MVP에 포함되지 않는 기능 (Post-MVP)

| 기능 | 이유 |
|---|---|
| 문서별 권한/공유 설정 | MVP는 로그인한 모든 사용자가 모든 문서 접근 가능 |
| 댓글, 제안 모드 | 복잡도 증가, 다음 단계에서 추가 |
| 버전 히스토리/되돌리기 | Yjs로 구현 가능하지만 UI 작업이 큼 |
| PDF / Word 내보내기 | MVP 범위 밖 |
| 이미지 업로드 | 파일 저장소 필요 |
| 모바일 반응형 최적화 | 데스크톱 기준으로 먼저 검증 |
| 본격적인 로그인 (OAuth) | 이름 입력만으로 충분 (MVP) |
| 문서 삭제/아카이브 | 필요 시 쉽게 추가 가능 |
| 실시간 제목 동기화 | 제목은 REST API로 저장 (옆 사람은 새로고침 시 반영) |

## 5. 성공 지표 (Success Criteria)

MVP가 "성공"했다고 말할 수 있으려면:

- [ ] 두 개의 브라우저 창에서 같은 문서를 열고, 한쪽에서 타이핑 시 반대쪽에 **1초 이내** 반영
- [ ] 두 사용자가 **동시에 같은 위치**에 타이핑해도 글자 손실 없음
- [ ] 서로의 커서가 **색깔과 이름**으로 명확히 구분됨
- [ ] 모든 창을 닫고 서버를 재시작한 뒤 문서를 열어도 **내용이 그대로 복원**됨
- [ ] 문서 목록 페이지에서 **마지막 수정 시간** 기준으로 정렬되어 표시됨

## 6. 사용자 시나리오 (User Stories)

### Story 1: 팀 회의록 작성
> Alice는 회의 시작 전에 "새 문서"를 만들어 제목을 "4월 정기 회의"로 바꾸고, 팀원들에게 URL을 공유한다. Bob과 Charlie가 들어와서 각자 맡은 주제를 **동시에** 작성한다. 회의가 끝나고 창을 닫아도 문서는 안전하게 저장된다.

### Story 2: 혼자 작성 후 공유
> Bob은 혼자 기획서를 먼저 작성한다. 나중에 Alice에게 URL을 보내 피드백을 받는다. Alice가 접속하면 Bob은 화면 상단에서 "Alice가 접속했네" 하고 알아차리고, 색깔 커서로 어디를 읽고 있는지 확인한다.

### Story 3: 동시 편집 스트레스 테스트
> 3명이 같은 문단을 동시에 편집해도 글자가 섞이거나 사라지지 않는다. Yjs CRDT가 모든 변경사항을 안전하게 병합한다.

## 7. 제약 사항 및 한계

### 기술적 제약
- **로컬 개발 환경 전용**: MVP는 `localhost`에서 실행되는 데모 수준
- **WebSocket 서버 인증 없음**: 공개 인터넷에 배포 금지 (다음 단계에서 토큰 기반 인증 추가 예정)
- **동시 접속 규모**: 수십 명 수준까지 정상 동작 (수천 명 규모는 별도 스케일링 필요)

### 범위 제약
- 문서 권한 관리 없음 (로그인한 모두가 모든 문서 접근)
- 모바일 UI 최적화 없음
- 이미지/첨부파일 없음
